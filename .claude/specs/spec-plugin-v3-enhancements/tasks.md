### T-3: Implement lib/checkpoint.sh — create_checkpoint and handle_checkpoint_recovery

- **Status**: completed
- **Wired**: no
- **Verified**: yes
- **Requirements**: US-2
- **Description**: Implement `scripts/lib/checkpoint.sh` as a sourceable bash library with two functions.

  `create_checkpoint(iteration, work_dir)`:
  - Change to `work_dir`
  - Check for uncommitted changes via `git status --porcelain`
  - If no changes, set `CHECKPOINT_SHA=""` and return (skip checkpoint, no commit created)
  - If changes exist, run `git add -A && git commit -m "checkpoint: pre-iteration <N>"`
  - Capture and export `CHECKPOINT_SHA` from the resulting commit

  `handle_checkpoint_recovery(exit_code, checkpoint_sha, iteration, work_dir)`:
  - If `exit_code` is 0 or `checkpoint_sha` is empty, do nothing and return
  - If `exit_code` is non-zero and `checkpoint_sha` is non-empty:
    - Change to `work_dir`
    - Run `git reset --hard "$checkpoint_sha"`
    - Print "Rolled back to checkpoint: pre-iteration <N>" to stderr
    - If `git reset --hard` itself fails, print a critical error directing the user to manually inspect the branch
- **Acceptance**:
  - Sourcing the file exports `create_checkpoint` and `handle_checkpoint_recovery`
  - With no uncommitted changes, `CHECKPOINT_SHA` is empty and no commit appears in `git log`
  - With changes, a commit with message `checkpoint: pre-iteration 1` exists and `CHECKPOINT_SHA` is non-empty
  - Recovery with a non-zero exit code runs `git reset --hard` and prints the rollback message to stderr
  - Recovery with exit code 0 does not run `git reset`
